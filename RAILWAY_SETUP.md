# Налаштування Railway Volume для збереження замовлень

## Проблема
Railway деплоїть з GitHub, але коли замовлення створюються через Telegram-бота, вони зберігаються локально в файл `orders.json`, який не синхронізується з GitHub. Тому на Railway залишається стара версія файлу.

## Рішення: Railway Volume

Railway Volume - це персистентне сховище, яке зберігає дані між деплоями.

### Крок 1: Створити Volume в Railway

1. Відкрийте ваш проект на Railway
2. Перейдіть в розділ **Volumes**
3. Натисніть **+ New Volume**
4. Введіть назву: `orders-data`
5. Встановіть Mount Path: `/data`
6. Натисніть **Create**

### Крок 2: Перезапустити сервіс

Після створення Volume перезапустіть ваш сервіс, щоб він підключився до Volume.

**Важливо:** Бот і сервер тепер запускаються разом через `start.sh`. Переконайтеся, що:
- Файл `start.sh` має права на виконання (chmod +x)
- В Railway встановлено змінну середовища `BOT_TOKEN` з токеном вашого Telegram-бота

### Крок 3: Перевірити

Після перезапуску:
1. Створіть замовлення через Telegram-бота
2. Перевірте, чи воно з'являється на веб-сайті
3. Перезапустіть сервіс - замовлення має залишитися

## Альтернативне рішення: Змінна середовища

Якщо не хочете використовувати Volume, можете встановити змінну середовища:

1. В Railway перейдіть в **Variables**
2. Додайте змінну: `DATA_DIR=/tmp/data` (або інший шлях)
3. Перезапустіть сервіс

## Важливо

- Файл `orders.json` тепер зберігається в `/data/orders.json` на Railway
- Локально файл залишається в корені проекту
- При деплої з GitHub старий файл не перезаписує дані з Volume
